## 二、简答题

1）总时延通常包括发送时延、传播时延、处理时延、排队时延。光纤传输速率高并不是因为传播时延比铜线小很多；传输速率主要由带宽、编码、调制和设备能力决定，传播时延主要由传播速度和链路长度决定。

2）Hub 工作在物理层，不隔离冲突域，不隔离广播域。Switch 工作在数据链路层，每个端口可隔离冲突域，但默认不隔离广播域。

3）DNS 解析过程：主机查本机缓存和 hosts；没有则向本地域名服务器查询；本地域名服务器递归或迭代查询根域名服务器、顶级域名服务器、权威域名服务器；获得 IP 后返回客户端并缓存。

4）超时后：

```text
ssthresh = 36 / 2 = 18
cwnd = 1 MSS
```

随后进入慢开始阶段。

5）CSMA/CD 中 k=2：

```text
r ∈ [0, 2^2 - 1] = [0,3]
退避时间 = r * 2τ
```

等待时间可能是 0、2τ、4τ、6τ，范围 0 到 6τ。

6）原数据 `7E 5E 7D 8D 9D`。PPP 异步字节填充中 `7E -> 7D 5E`，`7D -> 7D 5D`，所以填充后为：

```text
7D 5E 5E 7D 5D 8D 9D
```

字符填充用于避免数据中的 7E 被误认为帧定界符，避免 7D 被误认为转义控制字节。

7）RIP 与相邻路由器交换路由表，内容主要是目的网络和距离，度量为跳数。OSPF 与同一区域内路由器交换链路状态信息，内容包括链路、代价、邻接关系等，再用 SPF 算法计算路由。

8）ICMP 请求穿过 NAT 前：

```text
源 IP = 192.168.26.16
目的 IP = 220.168.26.16
```

穿过 NAT 后：

```text
源 IP = 202.36.26.16
目的 IP = 220.168.26.16
```

返回时 NAT 根据映射把目的 IP 从 202.36.26.16 改回 192.168.26.16。

## 四、综合题

1）F 发来的 RIP 距离先加 1：Net1=3，Net2=5，Net3=6，Net4=16。

| 目的网络 | 下一跳 | 距离 | 原因 |
|---|---|---:|---|
| Net1 | F | 3 | 经 F 更短 |
| Net2 | B | 3 | 经 F 为 5，不优于原路由 |
| Net3 | F | 6 | 原下一跳就是 F，按 F 的新通告更新 |
| Net4 | F | 16 | 16 表示不可达，通常不加入有效路由表；若列出则标不可达 |

2）R1 若没有 R2 的 MAC，会在通往 R2 的链路上广播 ARP 请求，R2 单播 ARP 响应，R1 缓存映射。

R1 发往 R2 的以太网帧中：

```text
源 IP = IP1
目的 IP = IP6
源 MAC = R1 出接口 MAC
目的 MAC = R2 入接口 MAC
```

IP 地址端到端不变，MAC 地址逐跳改变。

服务器传输层根据目的端口进行分用。FTP 通常用 TCP 21，HTTP 通常用 TCP 80，因此交给 FTP 进程还是 Web 进程由目的端口和套接字绑定关系决定。
