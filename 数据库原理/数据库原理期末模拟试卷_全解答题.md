### 第二题

建表应包含 Employee 主键 EmpNo，Club 主键 ClubNo 且 Manager 外键引用 Employee(EmpNo)，Participate 以 (EmpNo, ClubNo) 或 (EmpNo, ClubNo, PDate) 作主键，并分别外键引用 Employee、Club。
Manager_Details 通过 Club 与 Employee 在 Manager=EmpNo 上连接。
Participation_Details 通过 Participate 连接 Employee 与 Club。
篮球社/舞蹈团可用 IN、JOIN 或 EXISTS。
未参加任何社团可用 NOT EXISTS 或 LEFT JOIN IS NULL。
参加全部社团用双重 NOT EXISTS。
至少参加 001 参加过的一个社团：ClubNo IN (SELECT ClubNo FROM Participate WHERE EmpNo='001')。
每社团人数：Club LEFT JOIN Participate GROUP BY ClubNo,Cname，COUNT(DISTINCT EmpNo)。

### 第三题

主要 FD：StudentID -> StudentName；CourseID -> CourseName, TeacherID；TeacherID -> TeacherName, Office；(StudentID, CourseID) -> Grade。
候选码：(StudentID, CourseID)。
最高范式通常为 1NF：存在 StudentID -> StudentName、CourseID -> CourseName/TeacherID 等对复合候选码的部分依赖，不满足 2NF。
3NF 分解示例：Student(StudentID, StudentName)，Course(CourseID, CourseName, TeacherID)，Teacher(TeacherID, TeacherName, Office)，SC(StudentID, CourseID, Grade)。SC 的 StudentID、CourseID 分别作外键。

### 第四题

实体：Customer、Book、Publisher、Warehouse。联系：Publisher-Book 为 1:N；Customer-Book 购买为 M:N，联系属性 PurchaseDate、Quantity；Warehouse-Book 存放为 M:N，联系属性 StockQty。
关系模式：Customer(CustomerID, CName, Phone)，Publisher(PubID, PubName)，Book(BookID, Title, Price, PubID)，Warehouse(WID, Location)，Purchase(CustomerID, BookID, PurchaseDate, Quantity)，Stock(WID, BookID, StockQty)。
主键：各实体 ID；Purchase 可用 (CustomerID, BookID, PurchaseDate)；Stock 可用 (WID, BookID)。外键按联系引用两端实体。
若属性均原子且非主属性完全依赖主键、无明显传递依赖，通常可达 3NF。

### 第五题

GRANT SELECT ON emp TO WangMing; GRANT SELECT ON dept TO WangMing;
GRANT SELECT ON emp TO LiuXing; GRANT UPDATE(salary) ON emp TO LiuXing;
GRANT ALL PRIVILEGES ON emp TO AdminUser WITH GRANT OPTION; dept 同理。
视图示例：CREATE VIEW my_emp AS SELECT * FROM emp WHERE empno = CURRENT_USER; GRANT SELECT ON my_emp TO PUBLIC; 实际系统中可用用户-员工映射表。
ALTER TABLE emp ADD CONSTRAINT ck_emp_salary CHECK (salary >= 0); ALTER TABLE emp ADD CONSTRAINT fk_emp_dept FOREIGN KEY (deptno) REFERENCES dept(deptno);
触发器应体现 BEFORE INSERT OR UPDATE ON emp、FOR EACH ROW、检查 NEW.salary < 0 并 SIGNAL/RAISE ERROR。

### 第六题

ACID：原子性、一致性、隔离性、持续性。
丢失修改：后写覆盖先写；脏读：读到未提交后又回滚的数据；不可重复读：同一事务两次读同一数据结果不同。
X 锁用于写，排斥其他 S/X 锁；S 锁用于读，可与其他 S 锁兼容，但排斥 X 锁。
一级防丢失修改；二级在一级基础上防脏读；三级在一级基础上读锁保持到事务结束，防不可重复读。
脏读例：T1 写 A=100 未提交，T2 读 A=100，T1 回滚。用二级封锁协议，T1 持有 X 锁时 T2 不能加 S 锁读取。
