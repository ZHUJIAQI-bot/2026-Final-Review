# 数据库原理复习资料勘误

以下为对两个 Word 资料的核对结果。总体上，资料的考试重点判断与老师标注图、2024-2025 回忆版试卷高度一致；主要问题集中在拼写、个别 SQL 语法模板和少数概念表述。

## 需要修正

1. `Non-trival Functional Dependency` 应改为 `Non-trivial Functional Dependency`。
2. `CRATE TRIGGER` 应改为 `CREATE TRIGGER`。
3. 事务定义处写成 `BEGIN TRANSACTION; ... COMMIT; ROLLBACK;` 容易误导。一个事务结束时应二选一：成功提交用 `COMMIT`，失败撤销用 `ROLLBACK`，不能同时执行。
4. 参照完整性处“当主键仅有单个属性组成时，这个属性不能再充当外键引用其他表”不严谨。一个属性可以同时是本表主键和外键，例如 1:1 扩展表、弱实体转换表。更准确说法是：外键必须引用被参照表的候选码或主键；是否同时作为本表主键取决于业务约束。
5. `CREATE VIEW <视图名> [全部列名] AS <子查询>` 建议改为：`CREATE VIEW 视图名 [(列名列表)] AS 子查询 [WITH CHECK OPTION];`。列名列表要么省略，要么与查询结果列数一致。
6. 资料中 `3NF：如果 R 属于 1NF，且不存在非主属性对码的传递依赖` 属于简化表述。答题时建议写成：先满足 2NF，再不存在非主属性对候选码的传递依赖。
7. 自然连接表述建议补充：自然连接是按同名属性进行等值连接，并在结果中去除重复同名属性列；不是所有去重后的等值连接都叫自然连接。

## 可保留但答题时要更规范

1. “只考到 3NF，BCNF 和 4NF 无需重点记忆”与资料和老师标注基本一致，但若题目问“是否达到 BCNF”仍要能用一句话解释：BCNF 要求每个非平凡函数依赖的决定因素都是候选码。
2. 触发器语法不同 DBMS 有差异，考试写通用框架即可：触发时间、触发事件、触发表、行级/语句级、条件、动作体。
3. 员工只能查自己记录这类安全题，标准答案通常用视图实现；实际系统还可能需要用户-员工映射表，不必展开太深。

## 资料重点判断是否可靠

可靠。建议复习优先级：

1. SQL 综合题：建表、视图、连接、嵌套、全部/不存在、分组统计。
2. E-R 图与关系模式转换。
3. 函数依赖、候选码、最高范式、3NF 分解。
4. 关系代数，尤其“全部课程”这类除法语义。
5. 安全性、完整性、触发器。
6. ACID、并发异常、X/S 锁和三级封锁协议。
