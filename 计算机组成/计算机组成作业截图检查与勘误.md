# 计算机组成作业截图检查与勘误

> 结论：截图里的大部分解题思路是对的，可以作为复习材料使用。下面列出容易在考试中被扣分或需要注明前提的地方。

## 1. 4.29 Cache 平均访问时间

截图采用：

- h = 4800 / (4800 + 200) = 0.96
- ta = htc + (1-h)tm = 30 x 0.96 + 150 x 0.04 = 34.8 ns
- e = tc / ta = 30 / 34.8 = 86.2%
- 加速比 = 150 / 34.8 = 4.31，即速度约为原来的 4.31 倍，若说“提高了多少倍”，可写提高约 3.31 倍。

检查意见：按本课程 PPT 第 4 章 Cache-主存系统效率公式，截图写法正确。若别的教材把未命中访问时间理解为“先查 Cache 再访问主存”，则 ta = htc + (1-h)(tc+tm)，结果会不同。考试建议按老师 PPT 公式写，并在答案中说明采用该公式。

## 2. 4.30 组相联 Cache 地址格式

截图结论正确：

- 每块 128 字，块内地址 b = 7 位。
- 主存 4096 块，主存块号 m = 12 位。
- Cache 64 块，c = 6 位。
- 每组 4 块，r = 2；组数 16，q = 4 位。
- 四路组相联主存地址格式：主存字块标记 8 位 + 组地址 4 位 + 字块内地址 7 位。
- Cache 地址位数：c + b = 6 + 7 = 13 位。

## 3. 4.32 命中率与加速比

截图思路基本正确，但建议把精确值写完整：

- 90 个连续字覆盖 12 个块。
- 重复读取 8 次，共 720 次读操作。
- 第 1 遍产生 12 次强制失效，之后都命中。
- 命中次数 = 720 - 12 = 708。
- 命中率 = 708 / 720 = 98.33%，不要只写 98%。

关于速度：截图按“命中访问 Cache 为 T/6，失效访问主存为 T”的简化模型：

- 有 Cache 时间 = 708 x T/6 + 12T = 130T。
- 无 Cache 时间 = 720T。
- 加速比 = 720T / 130T = 5.54。
- 更严谨表述：速度变为原来的 5.54 倍，或“提高了 4.54 倍”。

注意：如果题目明确失效时需要把整个 8 字块逐字调入，失效代价可能不是 1T，应按题干另算。

## 4. 4.39 磁盘题

截图计算总体正确，建议统一单位：

- 容量 = 275 道 x 12288 B/道 x 4 面 = 13516800 B。
- 最高位密度在最小磁道：12288B / (230π mm) = 17 B/mm = 136 bit/mm。
- 最低位密度在最大磁道：最大直径 = 230 + 275/5 x 2 = 340 mm，12288B / (340π mm) ≈ 11.5 B/mm = 92 bit/mm。
- 数据传输率 = 12288B x 3000/60 = 614400 B/s。
- 平均旋转等待时间 = (1/50) / 2 = 10 ms。

## 5. 5.33 程序查询、中断、DMA 比较

截图内容可用，建议把适用对象写得更标准：

- 程序查询：适合低速、简单、实时性要求不高的设备；CPU 等待较多。
- 程序中断：适合中低速 I/O，CPU 与设备可一定程度并行。
- DMA：适合高速、成块数据传送，如磁盘、网卡等。

## 6. 6.14 浮点格式分配

截图中的字段分配“阶符 1、阶码 5、数符 1、尾数 25”符合题意：32 位总长，要求表示 ±6 万且保证最大精度。

答题时建议写清理由：

- 32768 = 2^15 < 60000 < 2^16 = 65536，因此阶码至少要能表示 16。
- 阶符 1 位，阶码数值部分取 5 位。
- 数符 1 位。
- 剩余 25 位给尾数以保证最大精度。

## 7. 7.8 寻址方式题

截图对直接、变址、相对等求有效地址的思路是对的。需要注意：

- 立即寻址时，操作数本身在指令地址字段中，严格说通常不计算有效地址 EA。
- 相对寻址一般用取完指令后的 PC 加位移量，而不是用指令起始地址直接相加。

## 8. 6.26 补码浮点加减

截图给的是步骤型答案，复习时重点记流程：转换机器数 -> 对阶 -> 尾数加减 -> 规格化 -> 舍入/判溢出。考试若遇到类似题，过程比只写最终机器数更重要。
