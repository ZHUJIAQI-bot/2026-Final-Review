## 一、信号量同步互斥（14 分）

某实验室有一个共享打印室，室内只有 1 台打印机和 2 个等待位置。男同学只能由男助教登记打印，女同学只能由女助教登记打印。学生“进入打印室并登记”等动作必须保持原子性；助教“叫号并登记”等动作也必须保持原子性。若等待位置已满，新来的学生离开。

已给信号量：`mutex`、`empty`、`fullM`、`fullF`。

1. 写出四个信号量的初值和物理意义。
2. 用 P/V 或 wait/signal 写出男助教、女助教、男学生、女学生四类进程的伪代码。

# 参考答案

## 一、参考答案

信号量：

- `mutex=1`：保证学生进屋登记、助教叫号登记等动作互斥，维持共享等待区状态一致。
- `empty=2`：表示空等待位置数量。
- `fullM=0`：表示等待男助教处理的男学生数量。
- `fullF=0`：表示等待女助教处理的女学生数量。

伪代码示例：

```text
男学生:
  wait(empty)
  wait(mutex)
  进入打印室并登记男生队列
  signal(mutex)
  signal(fullM)
  接受登记/打印

女学生:
  wait(empty)
  wait(mutex)
  进入打印室并登记女生队列
  signal(mutex)
  signal(fullF)
  接受登记/打印

男助教:
  while true:
    wait(fullM)
    wait(mutex)
    叫一名男学生
    signal(mutex)
    signal(empty)
    登记/打印

女助教:
  while true:
    wait(fullF)
    wait(mutex)
    叫一名女学生
    signal(mutex)
    signal(empty)
    登记/打印
```

若题目强调“满了立即离开”，也可在学生进程中先用非阻塞方式检查 `empty`，但普通 P/V 解答通常按阻塞等待写。

## 二、参考答案

FCFS：

- 时间轴：`0-4 J1 | 4-7 J2 | 7-13 J3 | 13-15 J4`
- 完成时间：J1=4，J2=7，J3=13，J4=15
- 周转时间：J1=4，J2=6，J3=11，J4=11

SJF 非抢占：

- 时间轴：`0-4 J1 | 4-6 J4 | 6-9 J2 | 9-15 J3`
- 完成时间：J1=4，J2=9，J3=15，J4=6
- 周转时间：J1=4，J2=8，J3=13，J4=2

RR，q=2：

- 时间轴：`0-2 J1 | 2-4 J2 | 4-6 J3 | 6-8 J1 | 8-10 J4 | 10-11 J2 | 11-13 J3 | 13-15 J3`
- 完成时间：J1=8，J2=11，J3=15，J4=10
- 周转时间：J1=8，J2=10，J3=13，J4=6

EDF，按已到达作业中截止时间最早者，非抢占口径：

- 时间轴：`0-4 J1 | 4-7 J2 | 7-9 J4 | 9-15 J3`
- 完成时间：J1=4，J2=7，J3=15，J4=9
- 周转时间：J1=4，J2=6，J3=13，J4=5

## 三、参考答案

Need = Max - Allocation：

| 进程 | Need |
|---|---|
| P0 | 0 0 0 0 |
| P1 | 0 7 5 0 |
| P2 | 1 0 0 2 |
| P3 | 0 0 2 0 |
| P4 | 0 6 4 2 |

P1 请求 `Request1=(0,4,2,0)`：

- `Request1 <= Need1`：`(0,4,2,0) <= (0,7,5,0)`，成立。
- `Request1 <= Available`：`(0,4,2,0) <= (1,5,2,0)`，成立。

试探分配后：

- `Available=(1,1,0,0)`
- `Allocation1=(1,4,2,0)`
- `Need1=(0,3,3,0)`

安全性检查：

| 步骤 | Work | 可完成进程 | 完成后 Work |
|---|---|---|---|
| 1 | 1 1 0 0 | P0 | 1 1 1 2 |
| 2 | 1 1 1 2 | P2 | 2 4 6 6 |
| 3 | 2 4 6 6 | P1 | 3 8 8 6 |
| 4 | 3 8 8 6 | P3 | 3 14 11 8 |
| 5 | 3 14 11 8 | P4 | 3 14 12 12 |

存在安全序列：`P0 -> P2 -> P1 -> P3 -> P4`，因此可立即分配。

保证不死锁资源数公式：`m*(n-1)+1`。本题 `m=5, n=2`，至少 `5*(2-1)+1=6` 个。

### 2. 分页地址转换

页面大小 `1KB=1024B`。

- `0x07A4 = 1956`，页号 `1`，页内位移 `932`，块号 `2`，物理地址 `2*1024+932=2980=0x0BA4`。
- `3120`，页号 `3`，页内位移 `48`，块号 `5`，物理地址 `5*1024+48=5168=0x1430`。
- `0x1288 = 4744`，页号 `4`，页表无页 4，缺页。

### 3. 分段地址转换

- `(0,320)`：`320 < 500`，合法，物理地址 `1000+320=1320`。
- `(1,120)`：`120 >= 120`，越界，不合法。
- `(2,75)`：`75 < 90`，合法，物理地址 `600+75=675`。
- `(3,701)`：`701 >= 700`，越界，不合法。

## 六、参考答案

每块经过磁头前的时间为 `25/5=5ms`，读一块 5ms，CPU 处理一块 5ms。

连续顺序 A、B、C、D、E：

- 读 A：0-5，处理 A：5-10。
- 处理 A 时错过 B，因此等下一圈到 B，再读 B。
- 之后每块都类似：读完后处理 5ms，会错过下一个连续块，需要等一圈。
- 总时间可写为：`读 A 5 + 处理 A 5 + 4*(等待 20 + 读 5 + 处理 5)`。

交替存放可用顺序：`A, C, E, B, D`。这样 CPU 处理一个块的 5ms 中，磁盘刚好转过一个物理块，下一个逻辑块出现在磁头下。

- 优化后总时间：`5*(读 5 + 处理 5)`。

## 七、参考答案

块大小 `1KB`，盘块号 `4B`，每个索引块可存索引项数：

`N = 1KB / 4B`

1. 各级可管理数据量：
   - 直接索引：`10 * 1KB`
   - 一级间接：`N * 1KB`
   - 二级间接：`N * N * 1KB`
   - 三级间接：`N * N * N * 1KB`
2. 最大文件长度：
   - `10 * 1KB + N * 1KB + N^2 * 1KB + N^3 * 1KB`
3. 块长 `512B`、块号 `4B` 时，设 `M = 512B / 4B`。按老师图二要求，只列算式，不写结果：
   - 二级索引可寻址最大文件长度：`M * M * 512B`
   - 三级索引可寻址最大文件长度：`M * M * M * 512B`

## 八、参考答案要点

1. 并行是同一时刻多个事件同时发生；并发是同一时间间隔内多个事件交替推进。多道程序在一个作业等待 I/O 时让其他作业使用 CPU，从而提高 CPU、I/O、内存利用率和系统吞吐量。
2. 四种 I/O 控制方式：程序查询/轮询、中断驱动、DMA、通道。中断方式每次数据传输仍需 CPU 参与较多；DMA 由 DMA 控制器直接在设备和内存之间批量传送，只在开始和结束时较多打扰 CPU。
3. SPOOLing 用磁盘上的输入井、输出井和内存缓冲区配合输入/输出进程，模拟脱机输入输出。多个进程把输出先放入输出井排队，物理设备仍独占，但逻辑上每个进程像拥有自己的设备。
4. 连续结构顺序访问快、支持直接定位，但要求连续空间，不利于动态增长，有外部碎片；链接结构无外部碎片、易增长，但随机访问差，指针损坏影响大；索引结构随机访问好、易扩展，但需要额外索引块空间。
