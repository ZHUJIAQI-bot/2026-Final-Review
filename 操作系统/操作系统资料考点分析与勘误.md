# 操作系统资料考点分析与勘误

## 老师范围和命题倾向

本次范围是第 1 章到第 8 章前两节。老师图上明确写出：解答题约 86 分，问答题约 14 分。解答题重点集中在信号量问题、调度算法、存储管理、虚拟存储、文件管理；第 3 章死锁中银行家算法一定会考；第 6 章看控制方式和缓存机制；第 8 章看前两节。

去年回忆卷覆盖得很贴近这次范围：PV 理发师问题、SJF/RR/EDF 调度、银行家算法、可变分区 FF/BF、分页地址转换、OPT/LRU 页面置换、磁盘交替存放、UNIX 混合索引、设备驱动、并发 I/O 甘特图、簇和扇区。

## 两份 Word 资料建议修正

1. “1 既不是素数也不是合数”不是操作系统知识点，像是混入的其他课程备注，复习 OS 时可以忽略。
2. “不饿能进行自我创建、唤醒、激活和终止”应改为“不能”。这处是明显错别字。
3. 同步信号量那段写“在需要此后执行的语句之后加上 wait(s)”容易误导。准确写法是：前驱动作执行后 `signal(s)`，后继动作执行前 `wait(s)`。
4. “RR、EDF、LLF 期末不考察，了解即可”不能完全照信。去年卷已经考了 RR 和 EDF，老师本次也只笼统写“调度算法”，所以 RR/EDF/LLF 至少要会按表算。
5. 银行家算法预分配条件应写成两步：`Request <= Need`，且 `Request <= Available`。资料中“是否在当前系统总资源范围之内”的说法不够精确。
6. 最坏适应 WF “最不容易产生外部碎片”说得过满。更稳妥：WF 每次从最大空闲区分配，查找快，能留下较大的剩余分区，但也可能形成碎片，不能绝对说最不容易。
7. 改进 Clock 不建议简化为“00、01、10、11 顺序扫描”。常见描述是先找 `(A=0,M=0)`，再找 `(A=0,M=1)`，扫描过程中会把访问位 A 置 0；答题时按课堂口径写清扫描轮次。
8. SPOOLing 中“一个文件仅存放某一个进程的输入输出设备”应理解为“一个井文件存放某一进程的输入/输出数据或作业信息”，不是存放设备。
9. 索引结构“索引结点占用内存空间较少”要加限定：相对 FAT 这类整表结构，索引方式通常只需装入相关索引块或索引结点。
10. 分段地址转换部分“段内地址大于或等于段长不合法”是正确的，因为段内地址从 0 开始。

## 必会大题模板

### PV/信号量

答题顺序：

1. 写信号量名称、初值、物理意义。
2. 找互斥：同一时刻只能一个进程访问的资源，用 `mutex=1`。
3. 找同步：谁必须先发生，谁才能继续。前驱后 `signal`，后继前 `wait`。
4. 写每类进程伪代码，注意“进入/叫号/取放”等复合动作的原子性。

常见扣分点：信号量初值错；把 `wait` 和 `signal` 位置写反；多个前驱关系混用同一个信号量；忘记释放互斥信号量。

### 调度算法

必须会画时间轴。常考：

- SJF：非抢占式，CPU 空闲时从已到达作业中选服务时间最短者。
- RR：按时间片轮转，未完成的作业回到队尾，新到达作业按到达时刻入队。
- EDF：最早截止时间优先，若题目未说明抢占，按老师/题干口径处理；去年卷更像表格计算题，关键是写清时间轴。
- LLF：松弛度 `L = 截止时间 - 剩余运行时间 - 当前时间`，松弛度越小优先级越高。

### 银行家算法

标准步骤：

1. 计算 `Need = Max - Allocation`。
2. 检查 `Request <= Need`，否则请求超过最大需求，拒绝。
3. 检查 `Request <= Available`，否则资源不足，让进程等待。
4. 试探分配：
   - `Available = Available - Request`
   - `Allocation_i = Allocation_i + Request`
   - `Need_i = Need_i - Request`
5. 安全性检查：令 `Work = Available`，找 `Need_i <= Work` 的进程，完成后 `Work = Work + Allocation_i`。
6. 能找到覆盖全部进程的安全序列则允许分配，否则回滚并拒绝。

### 存储管理

- 可变分区：按题目给出的算法维护空闲区表，最后写“起始地址 + 长度”。
- 分页：`页号 = 逻辑地址 div 页面大小`，`页内位移 = 逻辑地址 mod 页面大小`；查页表得到块号，`物理地址 = 块号 * 页面大小 + 页内位移`。
- 分段：先判断段号是否存在，再判断 `段内地址 < 段长`，合法时 `物理地址 = 段基址 + 段内地址`。

### 虚拟存储

页面置换题必须逐行写内存块状态，不要只写结论。OPT 向未来看；LRU 向过去看；FIFO 看进入内存先后；Clock 看访问位，改进 Clock 同时看访问位和修改位。

### 文件与磁盘

第 7、8 章大题多半是索引结构、文件存储结构、空闲块管理、簇/扇区关系。老师图二的多重索引题要求“只列算式不写结果”，模拟卷参考答案也按这个规则处理。
