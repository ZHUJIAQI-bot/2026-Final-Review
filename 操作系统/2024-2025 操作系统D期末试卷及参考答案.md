# 参考答案

## 一、同步互斥：理发店问题

信号量：

| 信号量 | 初值 | 含义 |
|---|---:|---|
| mutex | 1 | 保护“进店”和“叫人”等临界动作，保证原子性 |
| fulla | 0 | 已进入店内、等待男理发师服务的男顾客数 |
| fullb | 0 | 已进入店内、等待女理发师服务的女顾客数 |
| empty | 2 | 空闲理发椅数量 |

伪代码：

```text
男理发师:
while true:
    P(fulla)
    P(mutex)
    叫男顾客
    V(mutex)
    理发
    V(empty)

女理发师:
while true:
    P(fullb)
    P(mutex)
    叫女顾客
    V(mutex)
    理发
    V(empty)

男顾客:
P(empty)
P(mutex)
进店
V(fulla)
V(mutex)
接受理发

女顾客:
P(empty)
P(mutex)
进店
V(fullb)
V(mutex)
接受理发
```

## 三、银行家算法与死锁资源数

Need = Max - Allocation：

| 客户 | Need |
|---|---|
| 甲 | 0 0 0 0 |
| 乙 | 0 7 5 0 |
| 丙 | 1 0 0 2 |
| 丁 | 0 0 2 0 |
| 戊 | 0 6 4 2 |

乙请求 Request = (0,4,2,0)：

```text
Request <= Need乙 = (0,7,5,0)
Request <= Available = (1,5,2,0)
```

试分配后：

```text
Available' = (1,1,0,0)
Allocation乙' = (1,4,2,0)
Need乙' = (0,3,3,0)
```

安全性检查：

```text
Work = (1,1,0,0)
甲完成，Work = (1,1,1,2)
丙完成，Work = (2,4,6,6)
乙完成，Work = (3,8,8,6)
丁完成，Work = (3,14,11,8)
戊完成，Work = (3,14,12,12)
```

可立即满足。安全序列：甲 -> 丙 -> 乙 -> 丁 -> 戊。

单类资源防死锁公式：

```text
m >= n * (r - 1) + 1 = 5 * (2 - 1) + 1 = 6
```

至少需要 6 辆。

## 五、分页地址变换与页面置换

页面大小 1KB = 1024B = 0x400。

```text
0x0C5A: 页号 3，页内位移 0x05A，物理块 7，物理地址 0x1C5A
4700: 页号 4，页内位移 604，页号 4 不在页表，缺页
0x103C: 页号 4，页内位移 0x03C，页号 4 不在页表，缺页
```

访问序列 1,2,4,1,2,6,1,5,1，物理块数 3。

OPT 缺页次数：5 次。

```text
1缺, 2缺, 4缺, 1中, 2中, 6缺替4, 1中, 5缺替2或6, 1中
```

LRU 缺页次数：5 次。

```text
1缺, 2缺, 4缺, 1中, 2中, 6缺替4, 1中, 5缺替2, 1中
```

## 六、磁盘交替编号

连续存放 A、B、C、D：

```text
读A 0-5，处理A 5-10
错过B，等下一圈读B 25-30，处理B 30-35
错过C，等下一圈读C 50-55，处理C 55-60
错过D，等下一圈读D 75-80，处理D 80-85
```

总时间 85ms。

优化交替存放可采用物理顺序 A、C、B、D。

```text
读A 0-5，处理A 5-10
读B 10-15，处理B 15-20
等待C到来，读C 25-30，处理C 30-35
读D 35-40，处理D 40-45
```

题干写“读取并处理完”，答案为 45ms；若只统计最后一个块读入完成，则为 40ms。

## 七、UNIX 混合索引

一个索引块可存放 2048/4 = 512 个盘块号。

| 索引方式 | 可管理数据量 |
|---|---:|
| 10 个直接索引 | 10 * 2KB = 20KB |
| 一级间接索引 | 512 * 2KB = 1024KB = 1MB |
| 二级间接索引 | 512^2 * 2KB = 512MB |
| 三级间接索引 | 512^3 * 2KB = 256GB |

最长文件内容：

```text
10 * 2KB + 512 * 2KB + 512^2 * 2KB + 512^3 * 2KB
```

500MB 文件需要用到二级间接索引，不需要三级间接索引。

## 八、设备驱动、并发甘特图、簇与扇区

设备驱动程序是操作系统中直接控制和管理具体硬件设备的软件模块。作用包括屏蔽硬件差异、提供统一 I/O 接口、完成设备初始化、命令发送、数据传输、中断处理和错误处理等。

一种甘特图：

```text
输入设备: 0-2 P1 | 2-4 P2 | 4-6 P3
CPU:      2-5 P1 | 5-8 P2 | 8-11 P3
输出设备: 5-9 P1 | 9-13 P2 | 13-17 P3
```

全部完成时间 17ms。

文件大小 1026B，逻辑簇 1KB：

```text
ceil(1026 / 1024) = 2 个簇 = 2048B
2048B / 512B = 4 个扇区
```

最终分配物理存储空间 2048B。簇是文件系统分配空间的最小逻辑单位，扇区是磁盘物理读写的基本单位。
